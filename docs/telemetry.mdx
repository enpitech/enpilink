# Telemetry

**enpilink collects no telemetry.**

enpilink is an account-free, open fork. It has no analytics, no usage tracking,
and no metrics reporting. The CLI never phones home, and the server runtime
never emits counters to any remote endpoint.

There is nothing to opt out of:

- The CLI does **not** send any events to any analytics service.
- The server runtime does **not** emit any per-`tools/call` metrics.
- No hardcoded analytics key, endpoint, or IP exists anywhere in the codebase.
- No global config file is written to track preferences.

<Info>
**Your Privacy Matters**

enpilink never collects personal information, project code, file contents, tool
names, tool arguments, tool results, or any other data. Nothing leaves your
machine unless you explicitly configure it to (for example, by opening a public
tunnel).
</Info>

## Telemetry CLI commands

For compatibility, `enpilink telemetry status`, `enpilink telemetry enable`, and
`enpilink telemetry disable` still exist, but they are no-ops — there is no
telemetry to enable or disable.

```bash
enpilink telemetry status   # always reports: Disabled
```

## Background

Upstream Skybridge shipped CLI telemetry (PostHog) and a runtime DogStatsD
counter. enpilink removed both as part of decoupling from the original cloud
account. See the project [NOTICE](https://github.com/enpitech/enpilink/blob/main/NOTICE)
for attribution.
